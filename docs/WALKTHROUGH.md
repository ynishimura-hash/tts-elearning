## 1. Adalo からの移行

旧 Adalo システムの受講生を TTS 新システムへ一括移行する手順。

### 1-1. 事前準備
- `users` テーブルに既存ユーザー情報が登録済みであること
  - `email` / `full_name` 必須
  - `is_admin` / `is_online` / `is_free_user` で配信先を区別
- 環境変数
  - `GMAIL_USER` = `trademasternikkei225@gmail.com`
  - `GMAIL_APP_PASSWORD`（Google アプリパスワード、付録の手順参照）
  - `SUPABASE_SERVICE_ROLE_KEY` 設定済み
  - `NEXT_PUBLIC_APP_URL` （例: `https://tts-e.vercel.app`）

### 1-2. 仮パスワード一括発行
1. 管理者でログイン → サイドバー「メール配信」→ 「Adalo 移行」タブ
2. 配信対象を選択
   - 全員 / 対面のみ / オンラインのみ / 無料ユーザーのみ
3. 「仮パスワードを発行」ボタンをクリック
   - 12 桁のランダム英数字を全員分生成
   - Supabase Auth にユーザーが存在しなければ作成、存在すればパスワードを更新
4. 結果を CSV でダウンロード可能（バックアップ用）

### 1-3. ログイン案内メールを一斉配信
1. 「メール配信」タブに戻る
2. テンプレート選択：「Adalo からの移行案内」を選ぶと件名・本文が自動入力
3. 本文内の動的変数:
   - `{{full_name}}` 受講者名
   - `{{email}}` ログイン用メールアドレス
   - `{{temp_password}}` 仮パスワード
   - `{{login_url}}` ログイン URL
4. **テスト送信**で自分宛に確認 → **本番送信**

### 1-4. 案内テンプレート例

```
件名: 【TTS e-ラーニング】新システムへの移行のお知らせ

{{full_name}}様

いつもお世話になっております。
TTS e-ラーニングは新システムへ移行いたしました。
お手数ですが下記情報で再ログインをお願いいたします。

ログインURL: {{login_url}}
メールアドレス: {{email}}
仮パスワード: {{temp_password}}

※初回ログイン後、マイページからパスワードの変更をお願いします。
```

## 2. メール一斉配信

### 2-1. 画面構成
- 左：件名・本文・テンプレート選択
- 右：宛先リスト（CSV 取込 / 全ユーザー / 条件抽出）

### 2-2. 動的変数
本文・件名のいずれにも使える。受講者ごとに自動置換。

| 変数 | 置換内容 |
|------|---------|
| `{{full_name}}` | 受講者の氏名（`users.full_name`） |
| `{{email}}` | メールアドレス |
| `{{customer_id}}` | 顧客 ID |
| `{{temp_password}}` | 仮パスワード（移行モードのみ） |
| `{{login_url}}` | ログイン URL（環境変数 `NEXT_PUBLIC_APP_URL/login`） |

### 2-3. 配信フロー
1. 送信者を選択（運営アドレス）
2. 件名・本文を入力（変数を含めて OK）
3. 宛先を追加：
   - **CSV**: `email,name` 形式で取り込み
   - **TTS ユーザーから選択**: 一覧モーダルから複数選択
   - **手動追加**: メール＋名前を 1 件ずつ
4. プレビューで 1 件目の置換結果を確認
5. **テスト送信**で自分宛に確認
6. **送信**ボタン → 確認ダイアログ → 配信開始
7. 結果（成功/失敗/スキップ件数）が画面に表示
8. 配信履歴は `/admin/broadcasts/history` で参照可能

### 2-4. 配信停止
- メールフッターの「配信停止」リンクから受信者本人が停止できる
- 停止したアドレスは以降の一斉配信で自動スキップ

## 3. 個別相談の運用フロー

### 3-1. 受講生側
1. `/consultation`（または `/online/consultation`）から内部フォームで申込
2. 第一〜第三希望日時、相談内容を入力して送信
3. 管理者から確定通知が届くまで待つ

### 3-2. 管理者側
1. `/admin/consultations` 一覧で新着申込を確認
2. 詳細を開く
3. 確定方法（2 通り）：
   - **候補から選択**: 受講者の希望から 1 つ選んで確定
   - **直接入力**: 候補とは別の日時を直接入力して確定（4/15 追加）
4. 決済モーダルを開く → 入金確認後「決済完了」ボタンで確定
5. 自動でメール通知

## 4. Q&A 解説動画の紐付け

1. `/admin/qa` で Q&A を新規作成または編集
2. 「解説動画」セクションで「コース」を選択
3. 同コースのコンテンツ一覧から 1 つ選択
4. 公開すると、受講生の Q&A 画面で該当の質問にビデオアイコンが表示
5. 受講生がアイコンをクリックするとモーダルで動画再生

## 5. シミュレーション

### 5-1. 受講生の操作
1. `/simulation`（または `/online/simulation`）を開く
2. 入力項目：
   - **初期資金**: 万円単位、10 万円ステップ
   - **月利**: プリセット 10/20/30/40% から選択、または 1% 刻みで直接入力
   - **目標金額**: 達成までの予測月数を計算
3. グラフで複利推移を可視化
4. デフォルト期間：2 年（24 ヶ月）

### 5-2. 計算ロジック
- 単純複利： `n月後 = 初期資金 × (1 + 月利)^n`
- 目標金額達成時点を強調表示

## 6. ユーザー管理（コース別進捗確認）

1. `/admin/users` でユーザー一覧
2. ユーザーをクリックして詳細パネルを開く
3. **コース別進捗**セクション：
   - コース毎に「完了数 / 総数」と進捗バーを表示
   - クリックで展開すると、コース内のコンテンツ単位で完了状態と完了日時を確認可能
4. フィルタ：全員 / 対面 / オンライン / 無料 / 管理者 / テスト

## 付録: 開発者向け

### 環境変数チェックリスト
```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=

# メール配信（Gmail SMTP を優先、無ければ Resend にフォールバック）
GMAIL_USER=trademasternikkei225@gmail.com
GMAIL_APP_PASSWORD=                       # Google アカウント > セキュリティ > アプリパスワード で発行
GMAIL_SENDER_NAME=TTS e-ラーニング事務局   # 任意
RESEND_API_KEY=                           # フォールバック用（任意）

NEXT_PUBLIC_APP_URL=https://tts-e.vercel.app
CRON_SECRET=
LINE_CHANNEL_ACCESS_TOKEN=
```

### Gmail アプリパスワードの発行手順
1. Google アカウントにログイン → [https://myaccount.google.com/security](https://myaccount.google.com/security)
2. 「2 段階認証プロセス」を有効にする（必須）
3. 「アプリ パスワード」で新規発行 → アプリ名は任意（例: `TTS e-Learning`）
4. 表示された 16 桁のパスワードを `GMAIL_APP_PASSWORD` に設定

### Gmail SMTP の送信制限
- 無料 Gmail: **500 通／日**
- Google Workspace: **2,000 通／日**
- 受講者数が制限を超える場合は、複数日に分けて配信するか、Resend / SendGrid 等の専用配信サービスへ切替を検討
